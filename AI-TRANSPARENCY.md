# AI Transparency Notice

This document discloses how Artificial Intelligence (AI) and Large Language Models
(LLMs) are used in this project. It is part of our commitment to transparency,
accountability, and compliance with the EU AI Act and related regulations.

---

## 1. Human-in-the-Loop Guarantee

**All code in this project is human-initiated, human-written, and human-supervised.**

No part of this software was created autonomously by AI without human oversight.
Specifically:

- A human developer **initiates** every feature, bugfix, and design decision.
- A human developer **writes or co-writes** all source code.
- A human developer **reviews, tests, and approves** every change before it is merged.
- AI is used as a **tool under human direction** — never as an autonomous agent
  making unsupervised decisions.

This ensures compliance with the EU AI Act (Regulation (EU) 2024/1689),
in particular **Article 14 (Human Oversight)** and **Article 50 (Transparency
Obligations)**.

## 2. How AI Is Used in This Project

### 2.1 Development Assistance (Pair Programming)

AI tools are used as coding assistants for:

- Writing boilerplate and scaffolding code
- Generating and standardizing code comments and docstrings
- Code formatting and style unification
- Automated code reviews and quality checks
- Debugging assistance and error analysis
- Test generation

**All AI-generated or AI-assisted code is reviewed and validated by a human
developer before being committed.**

### 2.3 AI-Generated Media and Content

Any AI-generated media (images, diagrams, icons, audio, video, text content)
used in this project **must be clearly attributed** in the following format:

```
[Description]: Generated by [Model Name] ([Provider])
```

**Examples:**

- `logo.png`: Generated by DALL-E 3 (OpenAI)
- `architecture-diagram.svg`: Generated by Claude Opus 4.6 (Anthropic)
- `hero-image.png`: Generated by Stable Diffusion XL (Stability AI)
- `icon-set.svg`: Generated by Gemini Pro (Google DeepMind)

All AI-generated media credits are also listed in `CREDITS.txt` under the
section "AI-Generated Content Credits".

**Labeling requirement:** AI-generated images and media files should include
metadata or filename annotations (e.g., `_ai-generated` suffix) where
technically feasible, in accordance with EU AI Act Article 50(2).

## 4. What AI Does NOT Do in This Project

To be explicit about boundaries:

- AI does **not** make architectural or design decisions autonomously.
- AI does **not** commit, merge, or deploy code without human approval.
- AI does **not** have access to production systems or user data.
- AI does **not** replace human judgment on security, privacy, or compliance matters.
- AI output is **not** trusted blindly — it is always verified.

---

## 5. Compliance and Regulatory Notes

This project follows these principles in line with EU and international
AI governance frameworks:

- **Transparency**: This document and the `CREDITS.txt` file provide full
  disclosure of AI usage.
- **Human Oversight**: No AI-only decisions in the development lifecycle
  (EU AI Act Art. 14).
- **Labeling**: AI-generated content is labeled as such
  (EU AI Act Art. 50).
- **Accountability**: A human developer is always accountable for the
  final output.
- **Risk Assessment**: AI usage in this project is assessed as **minimal risk**
  under the EU AI Act risk classification, as it serves as a development tool
  under continuous human supervision.
